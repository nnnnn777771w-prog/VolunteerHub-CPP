# 🛡️ Volunteer Hub — Система координации штаба ЧС

Интеллектуальная консольная система на языке C++ с цветным GUI-интерфейсом. Программа создана для автоматизации процессов распределения ресурсов, учета волонтеров и мгновенной обработки экстренных заявок от населения в условиях чрезвычайных ситуаций.

---

## 🔑 Система авторизации и ролей

В приложение внедрено жесткое разграничение прав доступа для обеспечения безопасности персональных данных волонтеров и защиты базы данных от случайных или некорректных изменений оператора.

### Доступные режимы работы:
1. **Координатор (Администратор)** — полный доступ к управлению штабом (регистрация новых карточек, экстренный мэтчинг, закрытие задач, начисление часов и экспорт официальных отчетов).
   - **Логин:** admin
   - **Пароль:** 1234
2. **Гость** — режим ограниченного просмотра для внешних наблюдателей или смежных ведомств. Доступен только просмотр отсортированных таблиц ЧС, аналитики эффективности и встроенной инструкции. Все функции изменения базы данных в меню визуально заблокированы меткой (ЗАБЛОКИРОВАНО).

## 🚀 Ключевые возможности и алгоритмы

* **Экстренный мэтчинг (Алгоритм поиска):** Автоматическое посимвольное сопоставление категорий происшествия с навыками спасателей в реальном времени. Связывает задачу с нужным человеком за доли секунды.
* **Приоритет спасения жизней (Сортировка):** Реализация классического алгоритма пузырьковой сортировки (Bubble Sort) по убыванию уровня критичности задач. Самые экстренные вызовы (Уровень ЧС: 3) автоматически поднимаются на вершину таблицы.
* **Интуитивный цветной интерфейс:** Важные элементы интерфейса раскрашены с помощью WinAPI (SetConsoleTextAttribute):
  - **Ярко-красный:** Ошибки, заблокированные зоны и критические заявки уровня ЧС 3.
  - **Зеленый:** Успешные операции и уже выполненные (закрытые) задачи.
  - **Желтый / Голубой:** Навигация по меню, ID элементов и важная аналитика.
* **Smart-Валидация (Защита от дурака):**
  - Потоки ввода защищены от буквенного мусора (cin.clear(), cin.ignore()).
  - Строгий фильтр на текстовые категории (разрешены строго: Медицина, Поиск, Еда, Ремонт).
  - Автоматическая проверка формата ФИО волонтеров (обязательное использование нижнего подчеркивания _).
* **Аналитический модуль:** Расчет процента выполнения задач штаба и суммарного вклада работы волонтеров в человеко-часах.
* **Энергонезависимая БД:** Полноценное сохранение и автозагрузка данных через файловые потоки ifstream / ofstream (requests.txt, volunteers.txt).

## ⚠️ Решение проблем с отображением русского языка (Кодировка)

Поскольку на разных компьютерах установлены разные версии операционных систем (Windows 10/11) и разные терминалы, при первом запуске вместо кириллицы могут появиться иероглифы («кракозябры»).

Если вы столкнулись с этой проблемой, откройте файл main.cpp, найдите первые строчки в функции int main() и адаптируйте их под свою систему:

### Вариант 1: Стандартный для старых консолей Windows (уже в коде)
SetConsoleCP(1251);
SetConsoleOutputCP(1251);
setlocale(LC_ALL, "Russian");

### Вариант 2: Для современных терминалов Windows / VS Code (Unicode UTF-8)
Если у вас настроен современный терминал, замените кодировку 1251 на 65001:
SetConsoleCP(65001);
SetConsoleOutputCP(65001);
setlocale(LC_ALL, "Russian");

### Вариант 3: Универсальный (для Linux / macOS / CLion)
Если вы запускаете проект не на Windows, функции SetConsole... работать не будут (их нужно закомментировать). Оставьте только стандартную локализацию:
setlocale(LC_ALL, "ru_RU.UTF-8"); // или просто setlocale(LC_ALL, "Russian");
